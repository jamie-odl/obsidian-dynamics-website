SECURITY & TRUST

How We Protect Your Data

Operational intelligence demands operational security. Here's how Obsidian Dynamics protects the data we ingest, the intelligence we produce, and the systems our customers depend on.

COMPLIANCE

Standards & Certification

SOC 2 Compliant Infrastructure

All platform services run on SOC 2 Type II certified cloud infrastructure. This covers the physical hosting, network isolation, access controls, and monitoring layers that underpin SkyGrid, Strait-Signal, and RelayPoint. Our application-level controls are aligned to SOC 2 Trust Service Criteria across security, availability, and confidentiality.

UK GDPR & Data Protection Act 2018

Obsidian Dynamics Limited is a UK-registered data controller. We process data in accordance with the UK General Data Protection Regulation and the Data Protection Act 2018. Our privacy policy details what we collect, how we use it, and your rights as a data subject.

99.9% Uptime SLA

All production platform services — dashboards, APIs, webhook delivery, and SSE streaming — are covered by a 99.9% uptime service level agreement. Uptime is measured at the application layer, not just infrastructure. Planned maintenance windows are communicated 72 hours in advance.

INFRASTRUCTURE

Architecture & Data Residency

Hosting All production workloads run on SOC 2 Type II certified cloud infrastructure within UK and EU data centres.
Data Residency All customer data, event streams, and intelligence outputs are processed and stored within UK/EU infrastructure. No data leaves these jurisdictions.
Encryption in Transit All connections use TLS 1.2+ encryption. API endpoints, webhook delivery, SSE streams, and dashboard access are encrypted end-to-end.
Encryption at Rest All persistent data — event stores, baselines, configuration, and logs — is encrypted at rest using AES-256.
Network Isolation Production systems are network-isolated with private subnets. External access is restricted to authenticated API endpoints only.
Monitoring & Logging Centralised logging, anomaly detection on access patterns, and real-time alerting on infrastructure health. Audit logs retained for compliance.
ACCESS CONTROL

Authentication & Authorisation

API Authentication

All API access requires an API key plus HMAC request signing. Keys are scoped to specific data streams and rate-limited per contract. IP allowlisting is available on request for enterprise customers.

Dashboard Access

Dashboard authentication uses secure session management with HTTPS-only cookies. Session timeouts and re-authentication are enforced. Multi-factor authentication is available for enterprise accounts.

Webhook Delivery

Outbound webhooks include HMAC signature headers so receiving systems can verify payload authenticity. Delivery retries use exponential backoff. Failed deliveries are logged and accessible via API.

Access Reviews

API credentials are issued after use-case qualification. Active keys are reviewed periodically. Credentials can be rotated or revoked at any time without service interruption.

DATA HANDLING

What We Ingest, Store, and Deliver

Input Data

ADS-B transponder feeds (aviation) and AIS position reports (maritime) from authenticated receiver networks. No personally identifiable information is processed as part of intelligence production.

Intelligence Outputs

Structured JSON events — anomaly detections, chokepoint alerts, risk scores, and baseline metrics. Outputs are advisory and designed for analyst review, not automated decision-making.

Retention

Event data is retained for the duration of the customer contract plus a reasonable wind-down period. Contact form submissions are retained for 24 months. Customers can request data deletion at any time.

Third-Party Sharing

Customer data is never sold. Data is shared only with infrastructure service providers (bound by data processing agreements) and where required by law. No advertising, no analytics resale.
