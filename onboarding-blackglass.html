BLACKGLASS ONBOARDING

Blackglass setup from the repo

Blackglass ships as an open-source Next.js application. This page summarizes the entry path; authoritative steps live in the GitHub repository.

1. Source of truth

Clone blackglass-console and stay on main for current CI and release posture.

2. Local environment

Node 22.x, npm ci, copy .env.example to .env.local, then npm run dev. Optional Doppler helpers are documented upstream.

3. Quality gate

Before production changes, run npm run verify:stage0 (lint, OpenAPI parity, typecheck, unit tests, production build) as in CI.

DEPLOYMENT

Production orientation

Platform

DigitalOcean App Platform patterns, database migrations (direct Postgres port for Drizzle), and Stripe webhook secrets are covered in repository docs/ and .do/.

Auth and tenancy

Clerk sessions drive tenant context; requireTenantContext() and RLS helpers are the canonical enforcement points — never duplicate policy checks in ad-hoc handlers.
