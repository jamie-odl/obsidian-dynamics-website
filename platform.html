STRAIT SIGNAL FIREHOSE — LIVE

Maritime Event Stream

The Strait Signal firehose is operational. Obsidian Dynamics consumes maritime chokepoint events via three authenticated endpoints.

REST Polling

GET /api/firehose/events

Cursor-based event retrieval. Never misses events across reconnections. Supports cursor and limit parameters (max 1,000 per page).

cursor Integer — resume position (default: 0)
limit Integer — batch size, 1–1000 (default: 1000)

SSE Streaming

GET /api/firehose/stream

Real-time Server-Sent Events push. Events arrive the moment they are ingested. Includes heartbeat frames every 30 seconds for connection keep-alive.

Content-Type text/event-stream
Heartbeat 30s interval

Discovery

GET /api/firehose/meta

Metadata endpoint listing available event types, monitored locations, and the latest cursor position. Use this to bootstrap consumers or audit stream health.

event_types Array of available event categories
latest_cursor Current head position
SKYGRID FIREHOSE — PLANNED

Aviation Event Stream

The Project SkyGrid firehose will mirror the same API contract — cursor-based REST, SSE streaming, and meta discovery — for aircraft anomaly events across all eight detection classes.

REST Polling

GET /api/firehose/events

Same cursor-based contract as Strait Signal. Anomaly events indexed by type — loiter, ghost, squawk, rapid descent, ICAO spoof, GPS jamming, formation, callsign duplicate.

SSE Streaming

GET /api/firehose/stream

Real-time push of aircraft anomaly events. Designed for operations centres and SIEM integrations that need sub-second event delivery.

Discovery

GET /api/firehose/meta

Detection class catalogue, monitored airspace regions, current cursor head, and ingestion health metrics for consumer bootstrapping.
